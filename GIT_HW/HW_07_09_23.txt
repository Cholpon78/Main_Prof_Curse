1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
 git checkout имя файла.txt

2.Как вернуть файл из индекса в рабочий каталог?
git reset HEAD file.txt (можно написать нескольок имен файлов, если нужно вернуть несколько)

3.Как отменить ошибочный commit ?
git reset --hard HEAD~1 (это удаляет последний коммит, если надо еще дальше то пишем 2,3 и тд, в зависимости на скольок шагов назад надо удалить коммит)

4.Как исправить коммит отправленный на сервер GitHub?
git commit --amend -m "правильный комментарий"

5.Как создать новую ветку и переключится в нее?
создание - git checkout -b имя ветки
переключение git checkout имя ветки

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
Эти обе команды отвечают за слияние веток.  fast-forward merge - в этом случае изменения из одной ветки просто добавляются в новую ветку, без создания коммита.
non fast-forfard merge - а при этом слиянии создается новый коммитю В этом случае сохраняется история слияний.

6.Когда может возникнуть конфликт слияния веток, как  его разрешить? - 
Это может возникнуть, если были внесены изменения в один и тот же участок кода на обеих ветках. И GIT не знает какую версию нужно выбрать. 
Для этого надо открыть файлы в текстовом редакторе и там можно увидеть какие конфликты. Нужно убрать эти конфликты и снова добавить через команду git add и git commit  и git push

7.Можно ли удалить ветку? Как?  - Да ветку можно удалить с помощью команды git branch -d <имя-ветки>


В ответах на вопросы 1-4, 7 необходимо указать конкретные команда git. 
На 5 и 6 вопросы дать объяснение своими словами.
